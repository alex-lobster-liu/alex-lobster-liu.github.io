为什么你的ChatGPT越来越聪明：理解后训练中的RLHF和SFT

引言 你有没有想过，为什么现代AI助手（如ChatGPT）似乎能如此"理解"你的指令？秘密在于两种关键的后训练技术：基于人类反馈的强化学习（RLHF）和监督微调（SFT）。 但这里有一个惊人的真相：这两种技术并不像许多人假设的那样是独立的阶段。最新研究表明它们本质上是耦合的。 什么是RLHF和SFT？ 监督微调（SFT） 把SFT想象成"示范教学"。你给模型展示： 输入： “法国的首都是什么？” 理想输出： “法国的首都是巴黎。” 模型学习模仿这些模式。 基于人类反馈的强化学习（RLHF） RLHF更像"奖励教学"。你给模型展示两个回答： 回答A： “巴黎” ✓（好！获得奖励） 回答B： “伦敦” ✗（不好！受到惩罚） 模型通过试错来学习。 常见假设 大多数从业者（包括我自己直到最近）假设： 数据 → 预训练 → SFT → RL → 更好的模型 每个阶段独立地改进模型。应用SFT，变得更好。应用RL，变得更好。很简单，对吧？ 惊人的发现 最新研究挑战了这个假设。 在我们最新的论文“后训练中SFT与RL的非解耦性”中，我们证明： SFT和RL本质上是耦合的阶段，并非独立的。 这意味着什么 顺序很重要： 在RL之后应用SFT可能覆盖RL的改进 不是简单相加： 你不能简单地堆叠改进 流水线设计至关重要： 训练阶段的顺序和组合需要仔细考虑 为什么会这样 可以这样理解： SFT教模型遵循演示数据中的模式 RL教模型根据人类偏好最大化奖励 当你在RL之后应用SFT时： 模型可能忘记RL学到的行为 SFT模式可能与RL偏好冲突 模型回退到SFT风格的响应 实际影响 对AI从业者 不要假设独立性： 测试不同的训练顺序 监控后续阶段是否损害早期改进 考虑联合训练方法 流水线设计： ❌ 不好：预训练 → SFT → RL → SFT（回退RL） ✅ 好：预训练 → SFT → RL（到此为止） ✅ 更好：联合SFT+RL优化 对AI研究者 这开启了新的问题： ...
